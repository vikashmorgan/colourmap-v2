# Principles

## Engineering

- **Radical simplicity.** The simplest idea, not the first idea. If it can't be explained in one sentence, it's too complex.
- **No band-aids.** Root causes only. Fix it now or document why not.
- **Minimal blast radius.** Touch only what's necessary.
- **Test everything.** All code must be thoroughly tested — unit, integration, and e2e in relevant proportions. No untested code ships.

## Product

- **Cut features, not corners.** 3 things done perfectly beats 10 done adequately.
- **One way to do it.** Never two paths to the same outcome.
- **Invisible design.** If you're explaining how it works, it doesn't work.
- **Fewer states, fewer bugs.** Each state is surface area for bugs and confusion.
